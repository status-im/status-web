language: en
title: Modal verbs
---

Modal verbs are auxiliary verbs that modify the meaning of the main verb in a sentence. Modal verbs can be problematic in technical communication:

- Some modal verbs create a sense of uncertainty in readers.
- Using modal verbs like *should* or *must* makes your tone bossier and less helpful. In many cases, using the imperative is a better option. For example:

<Table>
  <TableHead>
    <TableHeader>Tone</TableHeader>
    <TableHeader>Example</TableHeader>
  </TableHead>
  <TableContent>
    <TableRow>
      <TableCell>**Assertive**</TableCell>
      <TableCell>Save your seed phrase in a secure location.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>Bossy</TableCell>
      <TableCell>You should save your seed phrase in a secure location.</TableCell>
    </TableRow>
  </TableContent>
</Table>

- Use this reference for the most frequent modal verbs:

  <TableHead>
    <TableHeader>Modal verb</TableHeader>
    <TableHeader>Usage</TableHeader>
  </TableHead>
  <TableContent>
    <TableRow>
      <TableCell>can</TableCell>
      <TableCell>Use it to express the possibility of the user to do something, like when you're introducing a new concept or functionality. Example: You can create multiple wallets for different purposes.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>should</TableCell>
      <TableCell>If possible, use the imperative form of the verb instead.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>have to</TableCell>
      <TableCell>Use *must* instead.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>must</TableCell>
      <TableCell>Use it very selectively and only when it's strictly necessary for the user to do something to avoid data or funds loss, data corruption, or unintended information disclosure.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>could, would, might</TableCell>
      <TableCell>Don't use.</TableCell>
    </TableRow>
