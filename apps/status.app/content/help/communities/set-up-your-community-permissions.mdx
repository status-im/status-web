In Status Communities, the [Owner and TokenMaster roles](./about-the-management-roles-in-status-communities.mdx) can customize community permissions using tokens. Tokens are digital assets in a blockchain (such as the [Ethereum blockchain](../getting-started/about-the-ethereum-blockchain.mdx)), including virtual coins, collectibles and ENS domain names.

Once you set up token-based permissions, certain actions within your community are only available to users who own a particular token. For instance, Owners can [restrict access to their community](./set-up-a-token-gated-community.mdx) or [delegate management functions](./delegate-management-functions-to-your-community.mdx) to specific token holders.

<Admonition type="info">
  In Status, your tokens are securely stored in your [Status Wallet](../wallet/status-wallet-your-quick-start-guide.mdx).
</Admonition>

## What to expect

- Using token-based permissions, community Owners and TokenMasters can restrict community access and actions to specific people based on token requirements.
- Only the community Owner and TokenMaster roles can add, [edit or delete community-level permissions](./permissions-by-role-in-status-communities.mdx).
- You set up community permissions like you [set up your channel permissions](set-up-your-channel-permissions.mdx). The only difference is the permissions you grant, either community-level (<ContextTag>Become admin</ContextTag> or <ContextTag>Become member</ContextTag>) or channel-level (<ContextTag>Moderate</ContextTag>, <ContextTag>View and post</ContextTag> or <ContextTag>View only</ContextTag>).
- Use the <ContextTag>Become member</ContextTag> permission to [restrict access to your community](./set-up-a-token-gated-community.mdx). Use the <ContextTag>Become admin</ContextTag> permission to [delegate management functions](./delegate-management-functions-to-your-community.mdx) to someone.

## Set up community permissions

Community Owners and admins customize permissions using three different variables: <ContextTag>Who holds</ContextTag> (optional), <ContextTag>Is allowed to</ContextTag> and <ContextTag>In</ContextTag>. For each variable, follow the same procedure using the table below for reference.

- If you turn off the <ContextTag>Who holds</ContextTag> option, the permission isn't bound to a token. This option only applies to channel-level permissions.
- For each permission, tokens are linked by <ContextTag>AND</ContextTag> operators. This means the new permission takes effect when the holder owns all the tokens. If you want to use the <ContextTag>OR</ContextTag> operator instead, create additional permissions.
- You can include up to five tokens in a single permission.
- You can create a maximum of five different <ContextTag>Become member</ContextTag> community-level permissions.
