21:T2691,<p>The <strong>Ethereum Dencun upgrade</strong>, activated on March 13, 2024, introduced <strong>EIP-4844</strong> (<a href="https://status.network/blog/what-is-eip-4844-proto-danksharding-l2-fees">proto-danksharding</a>). This created a new data type called "blobs" that Layer 2 rollups use to post transaction data back to Ethereum. Blobs are cheaper than traditional calldata. The result: L2 fees dropped by up to 10x overnight, making rollups far more viable for everyday use.</p><h2 id="what-changed-in-the-dencun-upgrade">What Changed in the Dencun Upgrade?</h2><p>Dencun combined two simultaneous upgrades. The execution layer upgrade was named "Cancun." The consensus layer upgrade was named "Deneb." Together they form "Dencun."</p><p>The upgrade included nine Ethereum Improvement Proposals (EIPs). <strong>EIP-4844</strong> was the headliner. It introduced <strong>blob-carrying transactions</strong>, a new transaction type purpose-built for rollup data.</p><p>Other EIPs in Dencun improved smart contract functionality. EIP-1153 added transient storage opcodes. EIP-6780 limited the SELFDESTRUCT opcode. But EIP-4844 drove the most visible impact.</p><h2 id="how-did-eip-4844-reduce-layer-2-fees">How Did EIP-4844 Reduce Layer 2 Fees?</h2><p>Before Dencun, rollups posted compressed data to Ethereum using calldata. Calldata lives permanently on-chain. It competes for block space with regular transactions. This made data posting expensive.</p><p>EIP-4844 created a separate data channel: blobs. Blobs have their own fee market. They do not compete with regular transactions for gas.</p><p>Each blob holds about 128 KB of data. Each Ethereum block can carry up to six blobs. This new capacity is reserved for rollups.</p><p>Blobs are temporary. Ethereum nodes prune blob data after roughly 18 days. This is enough time for rollups and other parties to store the data elsewhere.</p><p>Because blobs have a separate fee market and a dedicated supply, the cost of posting rollup data dropped dramatically.</p><h2 id="how-much-did-fees-actually-drop">How Much Did Fees Actually Drop?</h2><p>The impact was immediate and measurable. Several major rollups saw fee reductions of 90% or more within days of the upgrade.</p>

<p>These figures varied by transaction type and network congestion. The trend was consistent: posting data as blobs cost a fraction of what calldata required.</p><h2 id="why-temporary-data-storage-works">Why Temporary Data Storage Works</h2><p>Permanent data storage on Ethereum is expensive by design. Every full node must store calldata forever. This creates a growing burden on the network.</p><p>Blob data follows different rules. Nodes hold it for about 18 days. After that, the data is pruned from the consensus layer.</p><p>This works because rollups do not need Ethereum to store raw data forever. They need Ethereum to verify that data was available at the time of posting. This concept is called <strong>data availability</strong>.</p><p>Once the data has been verified and the rollup state is finalized, the raw data can live on cheaper storage layers. Archival services and rollup operators typically retain it.</p><h2 id="what-is-proto-danksharding">What Is Proto-Danksharding?</h2><p>EIP-4844 is called <strong>proto-danksharding</strong> because it lays the groundwork for full danksharding. Full danksharding will increase blob capacity from 6 per block to 64 or more.</p><p>Proto-danksharding introduced the blob transaction format. It introduced the separate fee market. It introduced KZG polynomial commitments for verifying blob data.</p><p>Full danksharding will add <strong>data availability sampling (DAS)</strong>. DAS lets nodes verify blobs without downloading the entire dataset. This is how Ethereum plans to scale to hundreds of blobs per block.</p><p>Proto-danksharding gave rollups immediate relief. Full danksharding will unlock far greater throughput in the future.</p><h2 id="what-does-this-mean-for-layer-2-networks">What Does This Mean for Layer 2 Networks?</h2><p>Cheaper data posting changes the economics of running an L2. Rollups spend less on their primary cost: settling data to Ethereum. This savings can flow to users as lower fees.</p><p>For rollups focused on high-volume, low-value transactions, Dencun was transformative. Social apps, games, and micropayment systems became economically viable on L2.</p><p>The fee reduction also opened design space for <a href="https://docs.status.network/general-info/gasless-transactions">gasless execution</a> models. When data costs drop, networks can explore alternative funding mechanisms that do not rely on per-transaction user fees.</p><h2 id="how-status-network-builds-on-this-foundation">How Status Network Builds on This Foundation</h2><p>Status Network is a gasless Ethereum Layer 2 implemented as a sovereign fork of the <strong>Linea zkEVM stack</strong>. It utilizes the <strong>blob-based data posting</strong> enabled by the <strong>Dencun</strong> upgrade to settle transaction data back to Ethereum efficiently. While other rollups use these cost savings to lower user fees, Status Network addresses a deeper structural failure: the <strong>spam auction</strong>, where MEV search bots consume the majority of block space on high-throughput chains while paying only a fraction of total fees.</p><p>The network replaces the traditional fee-market abstraction with a <strong>reputation-based execution model</strong> where users pay zero transaction fees for standard interactions. This model is powered by a <strong>native funding pool</strong> governed by the community. Rather than extracting fees from users, the network generates revenue from <strong>bridged yield</strong>, such as ETH staked via <strong>Lido V3 stVaults</strong> and stablecoins lent on <strong>Morpho</strong>, and protocol fees from native L2 applications like the <strong>Orvex DEX</strong> and <strong>FIRM CDP</strong>.</p><p><strong>Karma holders</strong> (the network's reputation earners) govern the allocation of this yield through on-chain voting. <strong>Dencun blobs</strong> and subsequent upgrades like <strong>Fusaka</strong> (December 2025) have made this model structurally sustainable. Because <strong>blobs</strong> reduced the primary operational cost of the network, settling transaction data to Ethereum, the native yield and application fee revenue can cover far more user transactions than was possible with traditional calldata.</p><h2 id="what-comes-after-dencun">What Comes After Dencun?</h2><p>Ethereum's next major upgrade, <strong>Pectra</strong>, continues the scaling roadmap. EIP-7691 (included in Pectra) increases the blob target from 3 to 6 per block and the maximum from 6 to 9.</p><p>Further ahead, <strong>PeerDAS</strong> will enable data availability sampling. This is the bridge between proto-danksharding and full danksharding.</p><p>Each upgrade reduces L2 data costs further.
