28:T2973,<p>A stealth address is a one-time cryptographic address generated fresh for every transaction. The sender creates it using the recipient's public key. Only the recipient can detect and spend funds sent to it. This breaks the on-chain link between sender and receiver, preventing anyone from mapping a user's full transaction history through blockchain explorers.</p><h2 id="why-are-public-blockchains-not-private">Why Are Public Blockchains Not Private?</h2><p>Every transaction on Ethereum is visible to everyone. Send ETH to a friend once, and that link is permanent. Blockchain explorers index every transfer. Analytics firms build identity graphs from these patterns.</p><p>This creates real problems. Employers can see salary payments. Merchants can see your full balance. Attackers can identify high-value targets.</p><p>Privacy is not about hiding wrongdoing. It is about preventing surveillance of routine financial activity.</p><h2 id="how-does-a-stealth-address-work">How Does a Stealth Address Work?</h2><p>The mechanism relies on public key cryptography. The process has four steps.</p><p><strong>Step 1: The recipient publishes a stealth meta-address.</strong> This is a special public key. It does not receive funds directly. Think of it as a mailbox number, not a street address.</p><p><strong>Step 2: The sender generates a one-time address.</strong> Using the recipient's stealth meta-address and a random number, the sender computes a fresh address. This address has never appeared on-chain before.</p><p><strong>Step 3: The sender publishes an announcement.</strong> A small piece of data goes to an on-chain announcement contract. This data helps the recipient (and only the recipient) find the transaction.</p><p><strong>Step 4: The recipient scans and claims.</strong> The recipient checks announcements using their private key. When they find a match, they derive the spending key for that one-time address.</p><p>The result: every payment lands at a unique address. No outside observer can connect two payments to the same person.</p><h2 id="what-is-erc-5564">What Is ERC-5564?</h2><p><a href="https://eips.ethereum.org/EIPS/eip-5564"><strong>ERC-5564</strong></a> is the Ethereum standard that defines stealth addresses. It specifies the announcement contract and the process for generating one-time addresses.</p><p>A companion standard, <strong>ERC-6538</strong>, handles the stealth meta-address registry. Users publish their meta-address once. Senders look it up to generate stealth addresses.</p><p>Together, these standards create an open protocol. Any wallet or dApp can implement stealth payments without custom infrastructure.</p><h2 id="how-do-stealth-addresses-differ-from-mixers">How Do Stealth Addresses Differ from Mixers?</h2><p>Mixers pool funds from many users and redistribute them. They break links by blending transactions together. But mixers require liquidity, create delays, and face regulatory scrutiny.</p><p>Stealth addresses work differently. No pooling occurs. No third party holds funds. The privacy comes from cryptography, not from mixing with other users.</p>
