27:T2638,<p><a href="https://github.com/ethereum/EIPs/blob/master/EIPS/eip-4844.md" rel="noreferrer"><strong>EIP-4844</strong></a>, activated in Ethereum's March 2024 Dencun upgrade, introduced <strong>blob transactions</strong>: a new data type that lets Layer 2 rollups post data to Ethereum at a fraction of the previous cost. By creating a separate fee market for temporary data blobs, proto-danksharding cut L2 transaction fees by up to 10x and laid the groundwork for full danksharding.</p><h2 id="why-did-layer-2-fees-need-fixing">Why Did Layer 2 Fees Need Fixing?</h2><p>Rollups bundle user transactions off-chain. They then post compressed proof data back to Ethereum.</p><p>Before EIP-4844, rollups stored this data in <strong>calldata</strong>. Calldata lives on-chain forever. Every Ethereum node must download and store it.</p><p>This permanence is expensive. Rollups competed with DeFi trades, NFT mints, and token transfers for the same block space.</p><p>When Ethereum got busy, L2 fees spiked. Users paying $0.10 on a quiet day might pay $2.00 during a surge.</p><p>The core problem: rollup data does not need to be permanent. Validators only need it long enough to verify proofs. Storing it forever wastes resources and inflates costs.</p><h2 id="how-blob-transactions-work">How Blob Transactions Work</h2><p>EIP-4844 solves this with a new transaction type: <strong>Type-3 transactions</strong> carrying data <strong>blobs</strong>.</p><h3 id="what-is-a-blob">What is a blob?</h3><p>A blob is a ~128 KB chunk of binary data. It attaches to a transaction but does not enter the EVM execution layer.</p><p>Blobs are temporary. Ethereum nodes store them for roughly 18 days, then prune them. This is enough time for anyone to verify rollup proofs.</p><h3 id="the-separate-fee-market">The separate fee market</h3><p>Blobs have their own fee market, independent of regular gas. Each Ethereum block targets 3 blobs and allows a maximum of 6.</p><p>Blob fees follow <strong>EIP-1559 style pricing</strong>. When blob demand is low, fees drop near zero. When demand is high, fees rise to manage congestion.</p><p>This separation means a DeFi surge on Layer 1 no longer drags up L2 posting costs.</p><h3 id="kzg-commitments">KZG commitments</h3><p>Each blob includes a <strong>KZG polynomial commitment</strong>. This is a compact cryptographic proof that the blob data is valid.</p><p>KZG commitments let validators confirm blob integrity without reading the full 128 KB. They also prepare Ethereum for <strong>data availability sampling</strong> in future upgrades.</p><h2 id="what-changed-after-dencun">What Changed After Dencun?</h2><p>The results were immediate and dramatic.</p>

This enables far more blobs per block.</li><li><strong>Expanded blob count</strong>: The target may grow from 3 to 32 or more blobs per block.</li><li><strong>Peer-to-peer blob distribution</strong>: A dedicated networking layer for blob propagation.</li></ul><p>Full danksharding could reduce L2 costs by another order of magnitude. Proto-danksharding built the transaction format and commitment scheme that make this possible.</p><h2 id="why-proto-danksharding-matters-for-gasless-networks">Why Proto-Danksharding Matters for Gasless Networks</h2><p>Cheaper data posting changes what is economically viable on Layer 2.</p><p>Before EIP-4844, subsidizing user gas fees required large treasuries. The math was hard to sustain. Every transaction carried a real calldata cost that someone had to pay.</p><p>With blobs, the base cost of posting rollup data approaches zero during low-demand periods. This makes alternative fee models practical.</p><p><a href="https://status.network/"><strong>Status Network</strong></a> takes this further. As a gasless Ethereum Layer 2 built on the <a href="https://linea.build/"><strong>Linea</strong></a> zkEVM stack, it removes gas fees entirely for users. Instead of charging per transaction, it funds operations through <a href="https://docs.status.network/tokenomics/economic-model"><strong>native yield</strong></a> from bridged assets and fees from ecosystem apps like <strong>Orvex</strong>.</p><p>Proto-danksharding did not create the gasless model. But it made the economics dramatically more favorable. When blob fees are near zero, the yield from productive capital (ETH staked via <a href="https://lido.fi/"><strong>Lido</strong></a>, stablecoins routed through lending protocols) can cover operational costs more easily.</p><p>Spam protection without gas fees requires a different mechanism. <strong>Status Network</strong> uses <strong>Rate Limiting Nullifiers (RLN)</strong>, a zero-knowledge protocol that replaces gas as the access control layer. <strong>Karma</strong>, a <a href="https://docs.status.network/tokenomics/karmic-tokenomics">soulbound reputation</a> token, determines each user's free transaction quota.</p><h2 id="the-road-ahead-peerdas-and-beyond">The Road Ahead: PeerDAS and Beyond</h2><p>Ethereum's next data availability upgrade is <strong>PeerDAS</strong> (Peer Data Availability Sampling). It will let validators confirm blob availability by sampling small portions instead of downloading entire blobs.</p><p>PeerDAS enables Ethereum to safely increase the blob target per block. More blobs per block means more rollup throughput at lower cost.</p><p>For L2 ecosystems, this creates a compounding effect. Each reduction in data costs expands the design space for new fee models, richer on-chain applications, and social or gaming use cases that require high transaction volumes.</p><h2 id="key-takeaways">Key Takeaways</h2><ul><li><strong>EIP-4844</strong> introduced blob transactions in Ethereum's Dencun upgrade (March 2024).</li><li>Blobs are temporary (~18 days), avoiding permanent calldata storage costs.</li><li>A separate blob fee market decoupled L2 costs from L1 congestion.</li><li>L2 fees dropped up to 10x immediately after activation.</li><li>Proto-danksharding is phase one.
